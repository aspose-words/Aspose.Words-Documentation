PDF是一种固定的页面格式,在用户中很受欢迎,并受到各种应用程序的广泛支持,因为PDF文档在任何设备上看起来都是一样的. 为此,转换为PDF是: Aspose.Words。 。 。 。

PDF本身是一种复杂的格式,因为它具有特定的文件结构,图形模型,字体嵌入,以及一些复杂的输出功能,如文件结构标记,加密,数字签名,和可编辑的形式. 此外,将一个文档转换为PDF需要几个计算阶段,这些阶段既复杂又耗时.

在本节中,我们将考虑与各种PDF标准文件合作时可能出现的主要问题,并描述解决这些问题的备选方案。

## 哪个 PDF 标准 Aspose.Words 支助

Aspose.Words 现在允许用户使用PDF/A-1、PDF/A-2和PDF/A-4格式以及PDF/UA-1:

* PDF/A-1具有透明度等严重限制,一些压缩选项被禁止
* PDF/A-2消除了PDF/A-1的一些限制,例如支持透明度和层效应或嵌入OpenType字体
* PDF/A-4假设订正符合水平:普通PDF/A-4符合水平相当于以前的版本的U级,A级符合水平被删除
* PDF/UA-1内容应按下列标准加贴标签和标准化: ISO 32000-1: 2008 (中文(简体) )

PDF/A是一个 ISO- 用于电子文件存档和长期储存的PDF标准化版本。 同时,PDF/UA是另一个 ISO 标准化的PDF版本,旨在确保使用辅助技术的残疾人无障碍。 要说明遵守PDF标准的程度,请使用 [Compliance](https://reference.aspose.com/words/java/com.aspose.words/pdfsaveoptions/#getCompliance) 属性。 由于存储条件,PDF/A文档必须嵌入所有字体并禁用加密,而PDF/UA只能嵌入所有字体.

在本节中,我们将仔细研究与PDF/A或PDF/UA-1文件的合作。
