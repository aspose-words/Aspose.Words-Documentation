Щоб краще ознайомитися з можливостями і функціональністю Aspose.Words, ми наводимо приклади, які можна завантажити з нашого репозиторію GitHub, Запустити і вивчити в деталях.

У цій статті ви можете ознайомитися з системними вимогами, а також з інформацією про те, як запускати приклади.

## Windows з пакетом Nuget

### Вимоги до програмного забезпечення та попередні умови

Будь ласка, переконайтеся, що ви відповідаєте наступним вимогам, перш ніж завантажувати та запускати приклади:

1. Visual Studio Код, Visual Studio 2022.
2. Встановлено менеджер пакетів NuGet та останню версію NuGet API для Visual Studio. (опціонально)
3. Виберіть опцію **nuget.org** у діалоговому вікні "Сервіс → Параметри" в розділі "NuGet менеджер пакетів → джерела пакетів".
4. Активне підключення до Інтернету необхідне для використання функції автоматичного відновлення пакета `NuGet` у проекті Examples. Якщо на комп'ютері, на якому повинні виконуватися приклади, немає активного підключення до Інтернету, дотримуйтесь інструкцій з Windows за допомогою Cmake пакета.

### Завантажте та запустіть приклади

Всі приклади Aspose.Words для C++ розміщені на [GitHub](https://github.com/aspose-words/Aspose.Words-for-C). Ви можете або клонувати сховище за допомогою улюбленого клієнта GitHub, або завантажити [файл ZIP](https://github.com/aspose-words/Aspose.Words-for-C/tree/master/Examples).

Отримавши копію сховища, ви можете виявити, що:

- Усі приклади знаходяться в папці **Examples**.
- В Visual Studio 2022 створені Visual Studio файлу рішень для C++.

Щоб запустити приклади, відкрийте файл рішення в Visual Studio і створіть проект:

- У прикладах **API Reference** структура базується на іменах класів, у прикладах **Docs** вона в основному базується на [Розробник обдурять](/words/cpp/developer-guide/) розділ документації.
- При першому запуску залежності будуть автоматично завантажені через NuGet.
- Папка **Data** у кореневій папці **Examples** містить вхідні файли, які використовувались у прикладах.
- Всі приклади можуть бути запущені у вигляді модульних тестів.

## Windows з пакетом CMake

### Вимоги до програмного забезпечення та попередні умови

Будь ласка, переконайтеся, що ви відповідаєте наступним вимогам, перш ніж завантажувати та запускати приклади:

1. Visual Studio Код, Visual Studio 2022.
2. [Cmake](https://cmake.org/download/) >= 3.16.0
3. Завантажте останній пакет CMake з https://downloads.aspose.com/words/cpp

### Завантажте та запустіть приклади

Всі приклади Aspose.Words для C++ розміщені на [GitHub](https://github.com/aspose-words/Aspose.Words-for-C). Ви можете або клонувати сховище за допомогою улюбленого клієнта GitHub, або завантажити [файл ZIP](https://github.com/aspose-words/Aspose.Words-for-C/tree/master/Examples).

Помістіть папки `Aspose.Words.Cpp` та `CodePorting.Native.Cs2Cpp_*` у корінь копії сховища.

Усі приклади знаходяться в папці **Examples**.

Щоб запустити приклади, виконайте наступні команди з кореневого каталогу копії сховища:
{{< highlight bash >}}
cmake -G "Visual Studio 17 2022" -Thost=x64 -Ax64 -S Examples\DocsExamples\ -B Examples\DocsExamples\build
{{< /highlight >}}

Рішення для Visual Studio буде створено в `Examples\DocsExamples\build`

Щоб запустити приклади, відкрийте файл рішення в Visual Studio і створіть проект:

- У прикладах **API Reference** структура базується на іменах класів, у прикладах **Docs** вона в основному базується на [Розробник обдурять](/words/cpp/developer-guide/) розділ документації.
- Папка **Data** у кореневій папці **Examples** містить вхідні файли, які використовувались у прикладах.
- Всі приклади можуть бути запущені у вигляді модульних тестів.

## Linux

### Вимоги до програмного забезпечення та попередні умови

Будь ласка, переконайтеся, що ви відповідаєте наступним вимогам, перш ніж завантажувати та запускати приклади:

1. CLang >= 3.9.1
2. [Ninja](https://ninja-build.org/)
3. [Cmake](https://cmake.org/download/) >= 3.16.0
4. Завантажте останній пакет CMake з https://downloads.aspose.com/words/cpp

### Завантажте та запустіть приклади

Всі приклади Aspose.Words для C++ розміщені на [GitHub](https://github.com/aspose-words/Aspose.Words-for-C). Ви можете або клонувати сховище за допомогою улюбленого клієнта GitHub, або завантажити [файл ZIP](https://github.com/aspose-words/Aspose.Words-for-C/tree/master/Examples).

Помістіть папки `Aspose.Words.Cpp` та `CodePorting.Native.Cs2Cpp_*` у корінь копії сховища.

Усі приклади знаходяться в папці **Examples**.

Щоб запустити приклади, виконайте наступні команди з кореневого каталогу копії сховища:
{{< highlight bash >}}
cmake -S Examples/DocsExamples -B Examples/DocsExamples/build -D CMAKE_BUILD_TYPE=Release
cmake --build Examples/DocsExamples/build

cmake -E chdir Examples/DocsExamples/build ctest --verbose
{{< /highlight >}}

- У прикладах **API Reference** структура базується на іменах класів, у прикладах **Docs** вона в основному базується на [Розробник обдурять](/words/cpp/developer-guide/) розділ документації.
- Папка **Data** у кореневій папці **Examples** містить вхідні файли, які використовувались у прикладах.
- Всі приклади можуть бути запущені у вигляді модульних тестів.
