Witamy w Aspose.Words zamiast Java!

Aspose.Words jest biblioteką klasy cross- platform, która umożliwia Twojej aplikacji wykonywanie różnorodnych zadań związanych z przetwarzaniem dokumentów. Nazywamy to Programowalnym Edytorem Dokumentów.

Stosowanie Aspose.Words, można tworzyć, modyfikować, konwertować, renderować i drukować dokumenty bez polegania na aplikacjach stron trzecich, takich jak Microsoft Word, lub Office Automation.

Warto zauważyć, że można przekonwertować dokument z dowolnego obsługiwanego formatu obciążenia do dowolnego obsługiwanego formatu zapisu, na przykład, Microsoft Word formaty do PDF, HTML lub Markdown, HTML do DOCX i tak dalej. Czytaj więcej o konwersji dokumentów w [Dokument Przelicza](/words/pl/java/convert-a-document/) sekcja.

Kolejną popularną opcją naszej biblioteki jest * Reports and Mails *. Nasi użytkownicy generują bardzo złożone indywidualnie dostosowane finansowe, medyczne i inne *Reports*. W ten sposób wygenerowany dokument wygląda, jakbyś ręcznie stworzył raport za pomocą zwykłego edytora tekstu. Kolejna opcja. *Mail*, zapewnia możliwość bardzo szybkiego generowania tysięcy i milionów prostych, ale nadal osobistych dokumentów opartych na tematyce.

W celu API tak jasne i oczywiste jak to możliwe, zdecydowaliśmy się rozpoznać i uhonorować wspólne Java praktyki rozwoju platformy. Tak więc, Aspose.Words zamiast Java przestrzega wytycznych kodowania powszechnie akceptowanych przez Java deweloperzy.

### Bogaty zestaw funkcji

Aspose.Words zamiast Java funkcje można podzielić na cztery główne obszary.

#### Konwersja

Zdolność łatwego i niezawodnego przetwarzania dokumentów jest jedną z głównych cech Aspose.Words.

Kompleksowa [Ładowanie, zapasowanie i konwertowanie](/words/pl/java/loading-saving-and-converting/) z [Obslugiwane formaty dokumentów](/words/pl/java/supported-document-formats/) pozwala użytkownikom konwertować dokumenty z [jeden format popularny](https://reference.aspose.com/words/java/com.aspose.words/loadformat/) do [another](https://reference.aspose.com/words/java/com.aspose.words/saveformat/), na przykład od DOCX do PDF, JPEG lub Markdown.

#### Document Object Model (DOM)

Program dostępu przez bogaty API do wszystkich elementów dokumentu i formatowanie pozwala na tworzenie, modyfikowanie, wyciąganie, kopiowanie, dzielenie, łączenie i zastępowanie zawartości dokumentu. Dowiedz się więcej w [Aspose.Words Document Object Model (DOM)](/words/pl/java/aspose-words-document-object-model/) sekcja.

#### Renderowanie

Używamy terminu [Renderowanie](/words/pl/java/rendering/) opis procesu konwersji dokumentu na format pliku lub medium, strony lub który ma pojęcie stron.

Możesz przekonwertować całe dokumenty lub konkretne strony do PDF lub XPS generowanie dokumentów po stronie serwera, drukowanie ich lub renderowanie stron. I to wszystko z wysoką wiernością - dokładnie jak Microsoft Word Zrobiłby to.

#### Sprawozdawczość

Generowanie dokumentów lub raportów od zera lub poprzez wypełnianie szablonów danymi ze źródeł danych lub obiektów biznesowych. Więcej informacji na temat raportowania i pobierania danych z baz danych można znaleźć w [Mail Merge i świętości](/words/java/mail-merge-and-reporting/) oraz [LINQ Silnik sprawozłszy](/words/java/linq-reporting-engine/) sekcje.

### Niezależność platformy

Aspose.Words zamiast Java obejmuje wszystkie Java środowiska rozwojowe i platformy wdrożeniowe. Nasze API mogą być wykorzystywane do opracowywania aplikacji dla szerokiego zakresu systemów operacyjnych, takich jak Windows, Linux, Mac OS, Android, i różne platformy.

Mówiąc prosto, jeśli platforma może działać standardowo <span notrans="<span notrans=" Java"=""></span>", > może działać Aspose.Words zamiast Java. Jeśli aplikacja może wywołać standard Java <span notrans="<span notrans=" API"=""></span>", > nie będziesz miał żadnych problemów, aby zadzwonić Aspose.Words zamiast Java API Z twojej aplikacji.

Obecnie naszym punktem odniesienia jest JDK1.7 i nowsza. Ale nadal popieramy JDK1.6.

Mamy trzy różne wydania oparte na tej samej bazie kodu:

* Aspose.Words zamiast Java
* Aspose.Words zamiast Android przez Java
* Aspose.Words zamiast JasperReports

Podczas gdy Java oraz Android wydania są opracowywane dla odpowiedniej platformy, Aspose.Words zamiast JasperReports to tylko grube opakowanie wokół "ordinal" Aspose.Words zamiast Java. To tylko dla wygody. Można również ręcznie zintegrować swoje JasperReports z Ustaw słowa - w szczególny sposób, który lepiej odpowiada Twoim potrzebom.

### Niezależność od innych wniosków

Aspose.Words nie wymaga aplikacji stron trzecich, na przykład, Microsoft Biuro, do zainstalowania na maszynie w celu pracy. Wszystkie komponenty pozycji są całkowicie niezależne. To czyni Aspose.Words doskonałą alternatywę dla automatyzacji pod względem bezpieczeństwa, stabilności, skalowalności / szybkości, ceny i funkcji do pracy z dokumentami i związanych z nimi zadań.

### Wydajność i skalowalność

Aspose.Words jest zaprojektowany do wykonywania znacznie na serwerze lub kliencie. Aspose.Words jest pojedynczym Java słoik, który może być zastosowany z dowolnym Java aplikacja po prostu ją kopiuje. Można również dodać `HarfBuzz` plugin dla zaawansowanych Funkcje czcionki, jeśli potrzebujesz (wysłane w oddzielnym słoiku). Standardowy Java ImageIO Rozszerzenie JAI można również dodać do platformy, aby wspierać dodatkowe ImageIO kodeki (nie trzeba w nowoczesnym Java środowiska).

Z Aspose.Words, można dosłownie wygenerować tysiące dokumentów w minutach, co obejmuje otwarcie dokumentu, modyfikację treści i formatowanie lub zaludnienie go z danymi, i zapisywanie go. Nawet najbardziej skomplikowane dokumenty są otwierane i zapisywane na maszynie P4 2.4Ghz 1Gb RAM w ułamkach sekundy.

Aspose.Words jest wielonitkowy bezpieczny tak długo, jak tylko jeden nitka działa na dokumencie na raz. Jest to typowy scenariusz, w którym jeden wątek pracuje nad jednym dokumentem. Różne wątki mogą bezpiecznie pracować na różnych dokumentach w tym samym czasie.
