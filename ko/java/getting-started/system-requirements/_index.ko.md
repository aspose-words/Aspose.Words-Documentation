## 선택 사항

위에서 언급 한대로 Aspose.Words 제품정보 Java 견적 요청 *only* 이름 * Java 실행 환경, 당신은 추가 라이브러리를 설치할 필요가 없습니다.

그러나 때로는 제 3 자 라이브러리를 추가해야 할 상황이 있습니다

- - - JogAmp JOGL 하드웨어 가속 OpenGL 렌더링을위한 라이브러리. 도서관은 aspose-words에 포함되어 있습니다. POM 외부 의존성으로 파일. 편집할 수 있습니다. POM 플랫폼에 필요한 JOGL 항아리를 보존하는 파일. 또는 하드웨어가 OpenGL 렌더링을 가속화하지 않으면 의존성을 제거 할 수 있습니다.
- - - HarfBuzz 폰트 엔진. JRE 9 및 더 새로운에 이미 포함되어 있습니다. JRE 8 이상을 사용하는 경우, 샤핑-harfbuzz-plugin 의존도를 사용할 수 있습니다. POM 파일. 그렇지 않으면 신뢰할 수 있습니다.
- - - Java Advanced Imaging (JAI) TIFF 이미지 지원 TIFF 이미지는 지원되지 않습니다. Java 8세 이상 이전의 TIFF 지원 Java 환경, Aspose.Words 제품정보 Java 에 따라 [Java Advanced Imaging (JAI) 제품 설명](https://www.oracle.com/java/technologies/java-archive-downloads-java-client-downloads.html) 이름 * Oracle· 아래 JAI를 설치하는 방법을 설명합니다.

### JAI 설치 방법 Windows

기본 JAI를 설치하기 위한 이 단계를 따르세요. ImageIO 으로 Windows::

1. 다운로드 **제이아이 1.1.3**·
   쓰기의 시간에, 설치기의 32 비트 버전 만 사용할 수 있습니다, 그래서 당신이 다운로드해야 JDK를 사용하는 경우 [Jai-1_1_3-lib-windows-i586-jdk.exe 파일 형식](https://download.java.net/media/jai/builds/release/1_1_3/jai-1_1_3-lib-windows-i586-jdk.exe) 다운로드해야 할 JRE를 사용하고 있다면 [Jai-1_1_3-lib-windows-i586-jre.exe 파일 형식](https://download.java.net/media/jai/builds/release/1_1_3/jai-1_1_3-lib-windows-i586-jre.exe)·
1. 명세 설치 프로그램을 실행하고 JDK/JRE가 설치되는 디렉토리에 해당합니다.
1. 다운로드 **JAI 이미지 I/O 1.1**·
   쓰기의 시간에, 설치기의 32 비트 버전 만 사용할 수 있습니다, 그래서 당신이 다운로드해야 JDK를 사용하는 경우 [ji_imageio-1_1-lib-windows-i586-jdk.exe를 사용하여](https://download.java.net/media/jai-imageio/builds/release/1.1/jai_imageio-1_1-lib-windows-i586-jdk.exe) 다운로드해야 할 JRE를 사용하고 있다면 [jai_imageio-1_1-lib-windows-i586-jre.exe 파일 형식](https://download.java.net/media/jai-imageio/builds/release/1.1/jai_imageio-1_1-lib-windows-i586-jre.exe)
1. 명세 설치 프로그램을 실행하고 JDK/JRE가 설치되는 디렉토리에 해당합니다.

### JAI 설치 방법 Linux

기본 JAI를 설치하기 위한 이 단계를 따르세요. ImageIO 으로 Linux::

1. 다운로드 **제이아이 1.1.3** 적절한 아키텍처 선택:
   1. 명세 [i586의](https://download.java.net/media/jai/builds/release/1_1_3/jai-1_1_3-lib-linux-i586.tar.gz) 32비트 시스템
   1. 명세 [힌디어64](https://download.java.net/media/jai/builds/release/1_1_3/jai-1_1_3-lib-linux-amd64.tar.gz) 64 비트의 경우 ( 인텔 프로세서를 사용하는 경우)
1. 명세 파일을 임시 디렉토리로 추출합니다.
1. 명세 지원하다 JAR JDK/JRE 파일 **라이브러리** 폴더.
1. 명세 JDK/JRE에 SO 파일을 이동 **오디오/amd64** 폴더.<br/>

   예를 들어, 64 비트에 Ubuntu 체계, 단계 1-4는 다음과 같이 보일 것입니다:<br/>
   {{< highlight csharp >}}
   $ cd /tmp
   $ wget https://download.java.net/media/jai/builds/release/1_1_3/jai-1_1_3-lib-linux-amd64.tar.gz && \
   gunzip -c jai-1_1_3-lib-linux-amd64.tar.gz | tar xf - && \
   mv /tmp/jai-1_1_3/lib/*.jar $JAVA_HOME/jre/lib/ext/ && \
   mv /tmp/jai-1_1_3/lib/*.so $JAVA_HOME/jre/lib/amd64/ && \
   rm /tmp/jai-1_1_3-lib-linux-amd64.tar.gz && \
   rm -r /tmp/jai-1_1_3
   {{< /highlight >}}
1. 다운로드 **JAI 이미지 I/O 1.1**, 적절한 아키텍처 선택:
   1. 명세 [i586의](https://download.java.net/media/jai-imageio/builds/release/1.1/jai_imageio-1_1-lib-linux-i586.tar.gz) 32비트 시스템
   1. 명세 [힌디어64](https://download.java.net/media/jai-imageio/builds/release/1.1/jai_imageio-1_1-lib-linux-amd64.tar.gz) 64 비트의 경우 ( 인텔 프로세서를 사용하는 경우)
1. 명세 파일을 임시 디렉토리로 추출합니다.
1. 명세 지원하다 JAR JDK/JRE 파일 **라이브러리** 폴더.
1. 명세 JDK/JRE에 SO 파일을 이동 **오디오/amd64** 폴더.<br/>

   예를 들어, 64 비트에 Ubuntu 체계, 단계 5-8는 다음과 같이 보일 것입니다:<br/>
   {{< highlight csharp >}}
   $ cd /tmp
   $ wget https://download.java.net/media/jai-imageio/builds/release/1.1/jai_imageio-1_1-lib-linux-amd64.tar.gz && \
   gunzip -c jai_imageio-1_1-lib-linux-amd64.tar.gz | tar xf - && \
   mv /tmp/jai_imageio-1_1/lib/*.jar $JAVA_HOME/jre/lib/ext/ && \
   mv /tmp/jai_imageio-1_1/lib/*.so $JAVA_HOME/jre/lib/amd64/ && \
   rm /tmp/jai_imageio-1_1-lib-linux-amd64.tar.gz && \
   rm -r /tmp/jai_imageio-1_1

## 임시 파일 폴더

Aspose.Words 제품정보 Java 컴퓨터의 임시 디렉토리를 사용하여 지정한 `java.io.tmpdir` 시스템 속성 Aspose.Words 제품정보 Java 임시 파일을 작성하고 렌더링이 완료되면이 파일을 제거합니다.

`java.io.tmpdir` 표준 Java 또한 디스크 기반 저장 정책에 의해 사용되는 시스템 속성. JVM이 임시 파일을 쓴 곳을 결정합니다.
