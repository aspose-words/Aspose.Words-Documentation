PDF는 사용자 중 매우 인기있는 고정 페이지 형식이며 PDF 문서와 같은 다양한 응용 프로그램에 의해 널리 지원됩니다. 이 이유를 들어, PDF로 변환하는 것은 중요한 기능입니다 Aspose.Words·

PDF는 특정 파일 구조, 그래픽 모델, 글꼴 embedding 및 문서 구조 태그, 암호화, 디지털 서명 및 편집 가능한 형태와 같은 복잡한 출력 기능이있는 것과 같은 복잡한 형식입니다. 또한, 문서를 PDF로 변환하는 것은 복잡하고 시간이 많이 걸리는 여러 계산 단계가 필요합니다.

이 섹션에서는 다양한 PDF 표준의 문서를 작업 할 때 발생 할 수있는 주요 문제를 고려하고 해결하기위한 옵션을 설명합니다.

## 어떤 PDF 제품 정보 Aspose.Words 지원하다

Aspose.Words 이제 사용자는 PDF/A-1, PDF/A-2 및 PDF/A-4 형식뿐만 아니라 PDF/UA-1로 작업할 수 있습니다

* PDF/A-1는 투명성과 몇몇 압축 선택권과 같은 심각한 한계가 금지됩니다
* PDF/A-2는 투명성과 층 효과의 지원과 같은 PDF/A-1의 한계의 일부를 삭제하거나 OpenType 글꼴의 embedding
* PDF/A-4는 개정된 준수 수준을 가정합니다. 일반 PDF/A-4 준수는 이전 버전의 U 준수와 동일하며, 레벨 A 준수가 제거됩니다
* PDF/UA-1 내용은 태그가 있어야 하고 표준화되어야 합니다 ISO 32000-1: 2008년

PDF/A는 ISO-PDF의 표준 버전은 전자 문서의 보관 및 장기 보관에 사용됩니다. 동시에 PDF/UA는 또 다른 ISO 지원 기술을 사용하는 장애인을 위한 접근성을 보장하기 위하여 디자인된 PDF의 표준화된 버전. PDF 표준 준수 수준을 지정하려면 [Compliance](https://reference.aspose.com/words/java/com.aspose.words/pdfsaveoptions/#getCompliance) 호텔 위치 저장 상태에 따라 PDF/A 문서는 모든 글꼴 및 비활성화 암호화를 포함해야 하며 PDF/UA는 모든 글꼴 만 포함해야 합니다.

이 섹션에서는 PDF/A 또는 PDF/UA-1 문서를 더 가까이 살펴볼 것입니다.

## 관련 기사 ISO PDF 파일 제품정보

다른 PDF 표준에 대해 자세히 알아 보려면 다음을 확인하십시오. ISO크기:

- PDF 1.7 = ISO-32000-1: 2008년
- PDF 2.0 = ISO-32000-2: 2020년
- PDF/A-1 = ISO-19005-1: 2005년
- PDF/A-2 = ISO-19005-2: 2011년
- PDF/A-4 = ISO-19005-4: 2020년
- PDF/UA-1 = ISO-14289: 2014년
