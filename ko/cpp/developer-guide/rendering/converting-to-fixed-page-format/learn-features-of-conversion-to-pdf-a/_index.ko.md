PDF 사용자들 사이에서 매우 인기가 널리 다양한 응용 프로그램에서 지원하는 고정 된 페이지 형식입니다 PDF 문서는 모든 장치에서 동일하게 보입니다. 이 때문에, PDF 의 중요한 기능입니다 Aspose.Words.

PDF 특정 파일 구조,그래픽 모델,글꼴 임베딩 및 문서 구조 태그,암호화,디지털 서명 및 편집 가능한 양식과 같은 복잡한 출력 기능을 가지고 있기 때문에 그 자체로 복잡한 형식입니다. 또한,문서를 PDF 복잡하고 시간이 많이 걸리는 여러 계산 단계가 필요합니다.

이 섹션에서는 다양한 문서 작업을 할 때 발생할 수있는 주요 문제를 고려할 것입니다 PDF 표준 및 해결 옵션을 설명합니다.

## 어떤 PDF 표준 Aspose.Words 지원

Aspose.Words 이제 사용자가 작업 할 수 있습니다 PDF/A-1, PDF/A-2 그리고 PDF/A-4 형식,그리고 PDF/UA-1:

* PDF/A-1 투명성과 같은 심각한 제한이 있으며 일부 압축 옵션은 금지되어 있습니다
* PDF/A-2 의 한계 중 일부를 제거 PDF/A-1 투명도 및 레이어 효과 지원 또는 OpenType 글꼴
* PDF/A-4 수정 된 적합성 수준을 가정합니다:일반 PDF/A-4 이 응용 프로그램은 사용자 정의 할 수 있습니다.
* PDF/UA-1 내용에 따라 태그를 지정하고 표준화해야 합니다. ISO 32000-1: 2008

PDF/A 이 ISO-표준화 된 버전 PDF 전자 문서의 보관 및 장기 보관에 사용하기위한 것입니다. 동시에, PDF/UA 또 다른 ISO 표준화 된 버전 PDF 보조 기술을 사용하는 장애인의 접근성을 보장하기 위해 설계되었습니다. 준수 수준을 지정하려면 PDF 표준,사용 [Compliance](https://reference.aspose.com/words/cpp/aspose.words.saving/pdfsaveoptions/get_compliance/) 재산. 저장 조건 때문에, PDF/A 문서는 모든 글꼴을 포함해야하며 암호화를 비활성화해야합니다. PDF/UA 모든 글꼴 만 포함해야합니다.

이 섹션에서는,우리는 작업에 대해 자세히 살펴 보겠습니다 PDF/A 또는 PDF/UA-1 문서.

## 관련 ISO 에 대한 PDF 표준

에 대한 자세한 내용은 다른 PDF 표준,다음을 확인 ISOs:

- PDF 1.7 = ISO-32000-1: 2008
- PDF 2.0 = ISO-32000-2: 2020
- PDF/A-1 = ISO-19005-1: 2005
- PDF/A-2 = ISO-19005-2: 2011
- PDF/A-4 = ISO-19005-4: 2020
- PDF/UA-1 = ISO-14289: 2014
