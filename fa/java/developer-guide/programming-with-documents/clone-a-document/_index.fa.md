شبیه سازی یک سند فرآیند ایجاد یک کپی یکسان از یک سند اصلی است که می تواند عملکرد را بهبود بخشد و شما را از نشت حافظه بالقوه نجات دهد.

این مقاله موارد اصلی استفاده از کلون کردن یک سند و نحوه ایجاد کلون سند با استفاده از Aspose.Words را توضیح می دهد.

## عملیات با اسناد شبیه سازی

عملیات کلون به شما اجازه می دهد تا روند ایجاد اسناد را سریع تر کنید زیرا نیازی به بارگذاری و تجزیه یک سند از یک فایل در هر زمان ندارید.

پس از ایجاد کلون سند خود، می توانید آن را ویرایش کرده و عملیات مختلفی را روی آن انجام دهید، به عنوان مثال، آن را با سند اصلی مقایسه کنید، آن را به سند دیگری اضافه یا وارد کنید. همچنین می توانید عناصر شبیه سازی شده یا محتوای آنها را قبل از قرار دادن آنها در یک سند دیگر تغییر دهید.

## ایجاد یک کلان سند

Aspose.Words به شما اجازه می دهد تا یک سند را با استفاده از روش [Clone](https://reference.aspose.com/words/java/com.aspose.words/document/#deepClone) که یک کپی عمیق از سند را انجام می دهد و آن را باز می گرداند، کلان کنید. به عبارت دیگر، یک نسخه کامل از DOM را دریافت خواهد کرد. روش `Clone` سرعت تولید اسناد را افزایش می دهد و شما فقط به یک خط کد برای دریافت یک کپی از سند خود نیاز دارید.

کلون کردن یک سند جدید با همان محتویات اصلی تولید می کند، اما با یک کپی منحصر به فرد از هر یک از سند اصلی [nodes](https://reference.aspose.com/words/java/com.aspose.words/node/). شما همچنین می توانید عملیات کلون را به یک گره سند با استفاده از روش گره [Clone](https://reference.aspose.com/words/java/com.aspose.words/node/#deepClone-boolean) اعمال کنید، که به شما اجازه می دهد تا گره های سند ترکیبی را با و بدون گره های فرزند خود تکرار کنید.
