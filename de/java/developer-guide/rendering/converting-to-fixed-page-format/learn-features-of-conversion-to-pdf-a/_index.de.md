PDF ist ein festes Seitenformat, das bei Benutzern sehr beliebt ist und von verschiedenen Anwendungen weitgehend unterstützt wird, da ein PDF -Dokument auf jedem Gerät gleich aussieht. Aus diesem Grund ist die Konvertierung in PDF ein wichtiges Merkmal von Aspose.Words.

PDF ist an sich ein komplexes Format, da es eine bestimmte Dateistruktur, ein grafisches Modell, eine Schrifteinbettung und einige komplexe Ausgabefunktionen wie Dokumentstruktur-Tags, Verschlüsselung, digitale Signaturen und bearbeitbare Formulare aufweist. Darüber hinaus erfordert die Konvertierung eines Dokuments in PDF mehrere Berechnungsschritte, die komplex und zeitaufwändig sind.

In diesem Abschnitt werden die Hauptprobleme betrachtet, die bei der Arbeit mit Dokumenten in verschiedenen PDF -Standards auftreten können, und Lösungsmöglichkeiten beschrieben.

## Welcher PDF Standard Aspose.Words unterstützt

Aspose.Words ermöglicht Benutzern jetzt die Arbeit mit den Formaten PDF/A-1, PDF/A-2 und PDF/A-4 sowie PDF/UA-1:

* PDF/A-1 hat schwerwiegende Einschränkungen wie Transparenz und einige Komprimierungsoptionen sind verboten
* PDF/A-2 beseitigt einige der Einschränkungen von PDF/A-1, z. B. die Unterstützung von Transparenz- und Ebeneneffekten oder das Einbetten von OpenType-Schriftarten
* PDF/A-4 geht von überarbeiteten Konformitätsstufen aus: Die reguläre PDF/A-4 -Konformität entspricht der Konformität der Stufe U früherer Versionen, und die Konformität der Stufe A wird entfernt
* PDF/UA-1 Inhalte sollten gemäß ISO 32000-1: 2008 gekennzeichnet und standardisiert sein

PDF/A ist eine ISO-standardisierte Version von PDF, die für die Archivierung und Langzeitspeicherung elektronischer Dokumente vorgesehen ist. Gleichzeitig ist PDF/UA eine weitere ISO standardisierte Version von PDF, die die Zugänglichkeit für Menschen mit Behinderungen gewährleistet, die unterstützende Technologien verwenden. Verwenden Sie die Eigenschaft [Compliance](https://reference.aspose.com/words/java/com.aspose.words/pdfsaveoptions/#getCompliance), um den Grad der Übereinstimmung mit PDF-Standards anzugeben. Aufgrund der Speicherbedingungen muss PDF/A Dokument alle Schriftarten einbetten und die Verschlüsselung deaktivieren, während PDF/UA nur alle Schriftarten einbetten muss.

In diesem Abschnitt werden wir uns die Arbeit mit PDF/A- oder PDF/UA-1-Dokumenten genauer ansehen.

## Relevant ISO für PDF Normen

Um mehr über die verschiedenen PDF-Standards zu erfahren, überprüfen Sie die folgenden ISOs:

- PDF 1.7 = ISO-32000-1: 2008
- PDF 2.0 = ISO-32000-2: 2020
- PDF/A-1 = ISO-19005-1: 2005
- PDF/A-2 = ISO-19005-2: 2011
- PDF/A-4 = ISO-19005-4: 2020
- PDF/UA-1 = ISO-14289: 2014
