PDF ist ein festes Seitenformat, das bei den Benutzern sehr beliebt ist und von verschiedenen Anwendungen weit unterstützt wird, da ein PDF-Dokument auf jedem Gerät gleich aussieht. Aus diesem Grund ist die Umwandlung in PDF ein wichtiges Merkmal von Aspose.Words.

PDF ist ein komplexes Format für sich, da es eine bestimmte Dateistruktur, grafisches Modell, Schrifteinbettung und eine komplexe Ausgabefunktionalität wie Dokumentenstruktur-Tags, Verschlüsselung, digitale Signaturen und editierbare Formulare aufweist. Darüber hinaus erfordert die Umwandlung eines Dokuments in PDF mehrere Berechnungsstufen, die komplex und zeitaufwendig sind.

In diesem Abschnitt werden wir die wichtigsten Probleme berücksichtigen, die bei der Arbeit mit Dokumenten in verschiedenen PDF-Standards auftreten können und Optionen zur Lösung beschreiben.

## Welches PDF Standard Aspose.Words Unterstützung

Aspose.Words Jetzt können Benutzer mit PDF/A-1, PDF/A-2 und PDF/A-4 Formaten sowie PDF/UA-1 arbeiten:

* PDF/A-1 hat ernsthafte Einschränkungen wie Transparenz und einige Kompressionsoptionen sind verboten
* PDF/A-2 eliminiert einige der Einschränkungen von PDF/A-1, wie Unterstützung von Transparenz und Schichteffekten oder Einbettung von OpenType Schriften
* PDF/A-4 nimmt revidierte Konformitätsstufen an: Die regelmäßige PDF/A-4 Konformität entspricht der bisherigen U-Konformität der Versionen und die Level A-Konformität wird entfernt
* PDF/UA-1 Inhalt sollte nach ISO 32000-1: 2008

PDF/A ist ein ISO-Standardversion von PDF zur Verwendung in der Archivierung und Langzeitspeicherung von elektronischen Dokumenten. Gleichzeitig ist PDF/UA ein weiterer ISO standardisierte Version von PDF, die darauf abzielt, die Zugänglichkeit für Menschen mit Behinderungen zu gewährleisten, die Assistenztechnologie nutzen. Um die Einhaltung von PDF-Standards festzulegen, verwenden Sie die [Compliance](https://reference.aspose.com/words/java/com.aspose.words/pdfsaveoptions/#getCompliance) Eigentum. Aufgrund der Speicherbedingungen muss das PDF/A-Dokument alle Schriftarten einbetten und die Verschlüsselung deaktivieren, während PDF/UA nur alle Schriftarten einbetten muss.

In diesem Abschnitt werden wir uns genauer mit PDF/A oder PDF/UA-1 Dokumenten beschäftigen.

## Relevante ISO für PDF Normen

Um mehr über die verschiedenen PDF-Standards zu erfahren, lesen Sie folgendes: ISOS:

- PDF 1,7 = ISO-32000-1: 2008
- PDF 2.0 = ISO-32000-2: 2020
- PDF/A-1 = ISO-19005-1: 2005
- PDF/A-2 = ISO-19005-2: 2011
- PDF/A-4 = ISO-19005-4: 2020
- PDF/UA-1 = ISO-14289: 2014
