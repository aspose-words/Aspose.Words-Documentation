PDFはユーザーの間で非常に人気があり、さまざまなアプリケーションで広くサポートされています。PDFドキュメントは任意のデバイスで同じように見えます。 そのため、PDFへの変換は重要な機能です Aspose.Wordsお問い合わせ

PDFは、特定のファイル構造、グラフィカルモデル、フォントの埋め込み、ドキュメント構造タグ、暗号化、デジタル署名、および編集可能なフォームなどの複雑な出力機能を備えたため、それ自体によって複雑な形式です。 また、文書をPDFに変換するには、複雑な時間と時間のかかる計算段階が必要です。

このセクションでは、さまざまなPDF規格の文書を扱う際に発生する可能性のある主要な問題を検討し、それらを解決するためのオプションを記述します。

## PDFファイル スタンダード Aspose.Words サポート

Aspose.Words PDF/A-1、PDF/A-2、PDF/A-4、PDF/UA-1、PDF/UA-1、PDF/UA-1、PDF/A-2、PDF/A-4、PDF/A-4、PDF/A-1、PDF/A-1、PDF/UA-1、PDF/A-1、PDF/A-2、PDF/A-4、PDF/A-4、PDF/A-2、PDF/A-4、およびPDF/A-1:

* ※PDF/A-1は、透明度や圧縮オプションなど、深刻な制限があります。
* PDF/A-2は、透明性とレイヤー効果のサポートやOpenTypeフォントの埋め込みなど、PDF/A-1の制限の一部を排除します。
* ※PDF/A-4は、修正された適合度を想定しています。通常のPDF/A-4適合性は、以前のバージョンのU適合度と同等であり、レベルA適合性は削除されます。
* PDF/UA-1 コンテンツは、タグ付けおよび規格化する必要があります ISO 32000-1: 2008年

PDF/A は ISO-電子文書のアーカイブおよび長期貯蔵の使用のために意図されているPDFの標準化された版。 同時に、PDF/UAは別のものです ISO 補助的な技術を使用する障害のある人々のためのアクセシビリティを確保するために設計されたPDFの標準化版。 PDF規格に準拠したレベルを指定するには、 [Compliance](https://reference.aspose.com/words/java/com.aspose.words/pdfsaveoptions/#getCompliance) 宿泊施設 保存条件により、PDF/A 文書はすべてのフォントを埋め、暗号化を無効にしなければなりません。PDF/UA はすべてのフォントを埋め込む必要があります。

このセクションでは、PDF/A や PDF/UA-1 のドキュメントを詳しく見ていきます。
