PDF é um formato de página fixa que é muito popular entre os utilizadores e é amplamente suportado por várias aplicações, uma vez que um documento PDF tem o mesmo aspeto em qualquer dispositivo. Por esta razão, converter para PDF é uma característica importante de Aspose.Words.

PDF é um formato complexo por si só, pois possui uma estrutura de arquivo específica, modelo gráfico, incorporação de fontes e algumas funcionalidades de saída complexas, como tags de estrutura de documentos, criptografia, assinaturas digitais e formulários editáveis. Além disso, a conversão de um documento para PDF requer várias etapas de cálculo, que são complexas e demoradas.

Nesta seção, consideraremos os principais problemas que podem surgir ao trabalhar com documentos em vários padrões PDF e descreveremos as opções para resolvê-los.

## Que PDF Padrão Aspose.Words Suporta

Aspose.Words agora permite que os usuários trabalhem com os formatos PDF/A-1, PDF/A-2 e PDF/A-4, bem como PDF/UA-1:

* PDF/A-1 tem limitações graves, como transparência e algumas opções de compressão são proibidas
* PDF/A-2 elimina algumas das limitações de PDF/A-1, como suporte a efeitos de transparência e camada ou incorporação de fontes OpenType
* PDF/A-4 assume níveis de Conformidade revistos: a conformidade regular PDF/A-4 é equivalente à conformidade do nível U das versões anteriores e a conformidade do nível A é removida
* PDF/UA-1 o conteúdo deve ser marcado e padronizado de acordo com ISO 32000-1: 2008

PDF/A é uma versão normalizada ISO-de PDF destinada a ser utilizada no arquivamento e armazenamento a longo prazo de documentos electrónicos. Ao mesmo tempo, PDF/UA é outra ISO versão padronizada de PDF projetada para garantir a acessibilidade para pessoas com deficiência que usam tecnologia assistiva. Para especificar o nível de conformidade com as normas PDF, utilize a propriedade [Compliance](https://reference.aspose.com/words/cpp/aspose.words.saving/pdfsaveoptions/get_compliance/). Devido às condições de armazenamento, PDF/A o documento deve incorporar todas as fontes e desativar a criptografia, enquanto PDF/UA deve incorporar apenas todas as fontes.

Nesta secção, examinaremos mais de perto o trabalho com documentos PDF/A ou PDF/UA-1.

## ISO relevante para as normas PDF

Para saber mais sobre os diferentes padrões PDF, Verifique o seguinte ISOs:

- PDF 1.7 = ISO-32000-1: 2008
- PDF 2.0 = ISO-32000-2: 2020
- PDF/A-1 = ISO-19005-1: 2005
- PDF/A-2 = ISO-19005-2: 2011
- PDF/A-4 = ISO-19005-4: 2020
- PDF/UA-1 = ISO-14289: 2014
