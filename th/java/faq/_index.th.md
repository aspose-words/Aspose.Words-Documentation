หน้า นี้ มี คํา ตอบ สําหรับ คํา ถาม ที่ ถาม บ่อย ที่ สุด. โปรด เรียน รู้ หน้า ปัจจุบัน – บาง ที นี่ อาจ ทํา ให้ คุณ สามารถ แก้ ปัญหา ของ คุณ ได้ อย่าง รวด เร็ว.

ถ้า คุณ ยัง ไม่ พบ คํา ตอบ สําหรับ คํา ถาม ของ คุณ คุณ อาจ ขอ คํา ตอบ ใน คํา ถาม นี้. สําหรับ ข้อมูล เกี่ยว กับ วิธี รายงาน คํา ถาม หรือ ปัญหา ของ คุณ อย่าง ถูก ต้อง โปรด ดู หนังสือ [การรองรับทางเทคนิค](/words/th/java/technical-support/) หน้า

คํา ถาม ที่ มี การ ถาม บ่อย ๆ ถูก จัด เป็น กลุ่ม โดย หัวข้อ เพื่อ ความ สะดวก สบาย ของ คุณ.

## ค่าทั่วไป

**Q: อะไร Aspose.Words?**

Aspose.Words สําหรับ Java เป็นไลบรารีสําหรับคลาสที่อนุญาตให้โปรแกรมของคุณ ทําการประมวลผลเอกสารได้หลากหลายงาน ด้วย Aspose.Words สําหรับ Javaคุณสามารถสร้าง, แก้ไข, แปลง, แปล, และพิมพ์เอกสารได้โดยไม่ต้องพิมพ์โปรแกรมส่วนที่สาม

รายละเอียดสามารถพบได้ในเอกสารของเรา

------

**Q: มันหมายความว่าอะไร"Aspose.Words ไม่ต้องการโปรแกรมที่ 3 เหรอ?**

Aspose.Words สําหรับ Java ไม่ต้องการที่จะติดตั้งโปรแกรมอื่น ๆ หรือแพกเกจโปรแกรมเพิ่มเติมที่จะทํางานได้ แค่ดาวน์โหลดและติดตั้ง Aspose.Words ตามที่อธิบายไว้ [การติดตั้ง](/words/th/java/installation/) บทความและเริ่มต้น

------

**Q: สิ่งที่แพลตฟอร์มทํา Aspose.Words ให้กําลังใจ?**

Aspose.Words สําหรับ Java ครอบ คลุม สภาวะ แวด ล้อม และ ระบบ การ ใช้ งาน ที่ นิยม กัน ส่วน ใหญ่. มัน API สามารถนําไปใช้ในการพัฒนาโปรแกรม สําหรับระบบปฏิบัติการขนาดใหญ่ Windows, Linux, และแมค OS และแพลตฟอร์มต่างๆ

สําหรับรายละเอียด ดู [Poduct Overview](/words/th/java/product-overview/) หน้าและ [แพลตฟอร์มและ Interopความสามารถ](/words/java/platforms-and-interoperability/) ส่วน

------

**Q: รูปแบบเอกสารอะไรที่ทําได้ Aspose.Words ให้กําลังใจ?**

Aspose.Words สําหรับ Java รองรับรูปแบบเอกสารที่นิยมมากที่สุด เช่น DOC, DOCX, HTML Markdown, PDF, XML และอื่น ๆ

คุณจะเห็นรายการของรูปแบบที่รองรับทั้งหมด [รูปแบบเอกสารที่รองรับ](/words/th/java/supported-document-formats/) หน้า

## กําลังแสดงเอกสาร

คุณสามารถหารายละเอียดเพิ่มเติมเกี่ยวกับการแสดงผลใน [การแสดงผล](/words/th/java/rendering/) แผนกเอกสาร

**Q: ทําไมผังและแบบอักษรในเอกสารผลลัพธ์จึงแตกต่างจากต้นฉบับ?**

แบบอักษรอาจจะไม่สามารถใช้ได้ หรือรูปแบบตัวอักษรอาจจะไม่รองรับ Microsoft Word แต่ไม่รองรับ Aspose.Words. Aspose.Words จะทํางาน [2553. 2013 Igricrayas language language language language (in อ383828-0) 743 changes changes, access access (2007) พ.ศ](/words/th/java/manipulate-and-substitute-truetype-fonts/).

เพื่อตรวจสอบว่าแบบอักษรหายไปหรือไม่:

- ใน Aspose.Words ใช้คําเตือนการแทนที่ตัวอักษร (สําหรับรายละเอียด โปรดดูที่ "วิธียอมรับว่าแบบอักษรถูกแทนที่" ส่วน [2553. 2013 Igricrayas language language language language (in อ383828-0) 743 changes changes, access access (2007) พ.ศ](/words/th/java/manipulate-and-substitute-truetype-fonts/) บทความ).
- ใน Microsoft Word เปิดกล่อง "การย่อยตัวพิมพ์ใหญ่"

------

**Q: ทําไมแบบอักษรไม่ล้มเหลวล่ะWindows เวทีและอักขระยูนิโค้ดจะถูกแทนที่ด้วย.... ไม่ป้องกัน glyph?**

เรามักจะเห็น `.notdef` glyph แทนที่จะเป็นตัวละครยูนิโค้ด ปกติจะเป็นกล่องหรือคําถาม glyph.

เพราะว่า Microsoft ตั้งค่าการล้มเหลวของสํานักงาน จะถูกเลือกเป็นค่าปริยายและ Microsoft ไม่พบแบบอักษรของสํานักงาน ผู้ใช้ต้องติดตั้ง Microsoft แบบอักษรสํานักงาน หรือเปลี่ยนการตั้งค่าการถอยกลับ

**Q: ทําไม จึง อาจ มี การ แสดง ข้อ ความ ที่ ซับ ซ้อน เช่น ภาษา ไทย หรือ ภาษา ฮีบรู ด้วย เหตุ ผล บาง ประการ เช่น ตําแหน่ง ที่ ไม่ ถูก ต้อง ใน เรื่อง ความ ไม่ สมดุล หรือ ความ ไม่ เสมอ ต้น เสมอ ปลาย?**

แบบอักษรของสคริปต์ที่ซับซ้อนบางอย่าง ต้องการคุณสมบัติแบบ TyPragraphy ขั้นสูง เพื่อใช้ในการประมวลผลเพื่อให้แสดงมันอย่างถูกต้อง รูปแบบการแปะภาพขั้นสูงถูกปิดการใช้งานใน Aspose.Words โดยปริยายแล้ว ผู้ใช้ต้องเปิดใช้งานชีวประวัติขั้นสูงด้วย [TextShaperFactory](https://reference.aspose.com/words/java/com.aspose.words/layoutoptions/#getTextShaperFactory).

**Q: ฉันต้องการเอกสารใบอนุญาตสําหรับตัวอักษรที่ฉันใช้ไหม**

ใช่ คุณรู้ ดังนั้น เมื่อทําการคัดลอกแบบอักษร โปรดจําไว้ว่าตัวอักษรส่วนใหญ่มีลิขสิทธิ์ เก็บบันทึกใบอนุญาตของแบบอักษรไว้ก่อน และตรวจสอบว่า สามารถโอนไปยังเครื่องอื่นได้อย่างอิสระ

## ที่ตั้งอยู่กึ่งกลาง Mail Merge

{{% alert color="primary" %}}

คุณสามารถหารายละเอียดเพิ่มเติมได้ mail merge ใน [Mail Merge และรายงาน](/words/java/mail-merge-and-reporting/) แผนกเอกสาร

{{% /alert %}}

**Q: ทําไม จึง ไม่ มี ทุ่ง นา รวม อยู่ ใน ผล ผลิต ที่ เกิด ขึ้น?**

ชื่อเดิมของสนามรวมยังคงเหมือนเดิม โดยไม่ต้องถูกแทนที่ด้วยข้อมูลที่จําเป็นจากแหล่งข้อมูล จากนั้น:

- ตรวจสอบข้อมูลที่ถูกโหลดอย่างถูกต้องลงในตาราง: ตั้งค่า [TableName](https://reference.aspose.com/words/java/com.aspose.words/imailmergedatasource/#getTableName) ถูก ต้อง กับ ทุก สิ่ง ที่ จําเป็น ต้อง มี กุญแจ สําคัญ และ ความ สัมพันธ์.
- ตรวจสอบว่าสนามรวมชื่อถูกต้อง ใช้ [GetFieldNames](https://reference.aspose.com/words/java/com.aspose.words/mailmerge/#getFieldNames) วิธีการในการรับชื่อในช่องข้อมูลทั้งหมด และตรวจสอบว่าชื่อของการรวมช่องข้อมูลในต้นแบบของคุณ ตรงกับชื่อที่อยู่ในแหล่งข้อมูลของคุณหรือไม่

------

**Q: เหตุ ใด การ ทํา รัง จึง ไม่ ได้ แสดง ข้อมูล จาก โต๊ะ เด็ก สําหรับ รายการ แรก ใน ตาราง บิดา มารดา แต่ แสดง รายการ ทุก อย่าง สําหรับ รายการ สุด ท้าย ใน โต๊ะ บิดา มารดา แม้ แต่ รายการ ที่ ไม่ เกี่ยว ข้อง กับ มัน จริง ๆ?**

นี่เพราะการรวมส่วนในแม่แบบไม่ได้ ก่อขึ้นอย่างถูกต้อง ซึ่งสามารถทําให้รังทั้งหมด mail merge ขอบเขตที่จะหยุดการแสดงทุกอย่าง เดอะ [StartTable](https://reference.aspose.com/words/java/com.aspose.words/documentbuilder/#startTable) การเปิดป้ายและ [EndTable](https://reference.aspose.com/words/java/com.aspose.words/documentbuilder/#endTable) ป้ายปิดจะต้องตรงกันและอยู่ในแถวหรือเซลล์เดียวกัน ตัว อย่าง เช่น ถ้า คุณ เริ่ม ทํา รัง เป็น รัง ใน เซลล์ ของ ตาราง คุณ ต้อง ทํา ให้ พื้น ที่ รวม เข้า ด้วย กัน ใน แถว เดียว กับ เซลล์ แรก.

------

**Q: ทําไม รายการ แต่ ละ รายการ จาก โต๊ะ บิดา มารดา จึง แสดง ทุก สิ่ง ใน โต๊ะ เด็ก แม้ แต่ รายการ ที่ ไม่ เกี่ยว ข้อง จริง ๆ?**

นี่ เป็น เพราะ ความ สัมพันธ์ ระหว่าง พ่อ แม่ กับ ลูก ไม่ ได้ ถูก ตั้ง ขึ้น หรือ ถูก กําหนด ไว้ อย่าง ผิด ๆ. มันจําเป็น

- ให้มั่นใจในความถูกต้องของข้อมูล **DataSet** และใช้ **DataRelation** วัตถุที่แสดงถึงความสัมพันธ์ระหว่างพ่อแม่กับลูก ระหว่างตารางข้อมูล
- ตรวจ สอบ ส่วน ที่ ชื่อ ว่า "วิธี ตั้ง ความ เกี่ยว พัน ระหว่าง ข้อมูล ใน รัง Mail Merge มีขอบเขต" ในบทความ [ที่ตั้งอยู่กึ่งกลาง Mail Merge ขอสาบานต่อมะลาอิกะฮฺที่จําแนกระหว่างความจริงกับความเท็จ](/words/java/nested-mail-merge-with-regions/).

------

**Q: เพราะ เหตุ ใด จึง เป็น ข้อ ยก เว้น: "บาป. การขยายภาพ: ไม่สามารถเปิดใช้งานเงื่อนไขนี้ได้ เนื่องจากไม่ใช่ค่าทั้งหมดที่มี ค่าแม่ที่เท่ากันกับค่าแม่" ขณะเรียกทํางานรัง mail merge?**

สิ่ง นี้ เกิด ขึ้น เพราะ ไม่ ใช่ ทุก คน ที่ มี บันทึก เกี่ยว กับ บิดา มารดา จะ มี บันทึก บุตร ดัง นั้น แหล่ง ข้อมูล ของ คุณ จึง ไม่ ตรง กับ เงื่อนไข ดัง ต่อ ไป นี้: * ทุก แถว ใน ตาราง บิดา มารดา ควร มี ความ สัมพันธ์ แบบ หนึ่ง ต่อ หนึ่ง ต่อ หนึ่ง กับ แถว ของ ตาราง เด็ก โดย อาศัย กุญแจ หลัก และ กุญแจ ต่าง ประเทศ *

ปิดการใช้ข้อกําหนดของกุญแจต่างประเทศ เมื่อคุณสร้าง **DataRelation**.

## แปลงเอกสาร

{{% alert color="primary" %}}

คุณสามารถหารายละเอียดเพิ่มเติมเกี่ยวกับการแปลงเอกสารใน [แปลงเอกสาร](/words/th/java/convert-a-document/) แผนกเอกสาร

{{% /alert %}}

**Q: จะแปลง DOCX เป็น PDF อย่างไร?**

ง่ายมากเช่นกัน แค่โหลดเอกสารไปยังโมเดล และบันทึกในรูปแบบที่รองรับ

ตัวอย่างรหัสต่อไปนี้ จะแสดงกระบวนการการแปลงแฟ้ม DOCX เป็น PDF:

{{< highlight java >}}
// Open the source DOCX document.
Document doc = new Document(dataDir + "Input.docx");

// Save the document to PDF format.
doc.save(dataDir + "output.pdf", SaveFormat.Pdf);
{{< /highlight >}}

## ทําเนียบกับเอกสาร

{{% alert color="primary" %}}

คุณสามารถหารายละเอียดเพิ่มเติมเกี่ยวกับโปรแกรมที่มีเอกสารใน [กําลังโปรแกรมด้วยเอกสาร](/words/th/java/programming-with-documents/) แผนกเอกสาร

{{% /alert %}}

**Q: จะแบ่งเอกสารทีละหน้าอย่างไร?**

Aspose.Words ให้คุณได้แยกเอกสารหลายหน้าทีละหน้า

ตัวอย่างรหัสต่อไปนี้ แสดงวิธีแบ่งเอกสารและบันทึกแต่ละหน้า เป็นเอกสารแยก:

{{< highlight java >}}
Document doc = new Document(dataDir + "TestFile (Split).docx");

int pageCount = doc.getPageCount();

// Save each page as a separate document.
for (int page = 0; page <= pageCount; page++)
{
	Document extractedPage = doc.extractPages(page, 1);
	extractedPage.save(dataDir + "SplitDocumentPageByPageOut_" + (page + 1) + ".docx");
}
{{< /highlight >}}

**Q: วิธีเปิดการเข้ารหัส** **เอกสาร?**

คุณสามารถลองเปิดเอกสารที่ถูกเข้ารหัสไว้โดยไม่มีรหัสผ่าน ซึ่งจะทําให้มีการยกเว้น

ตัวอย่างรหัสต่อไปนี้ จะแสดงวิธีเปิดเอกสารที่มีการเข้ารหัสด้วยรหัสผ่าน:

{{< highlight java >}}
// Create a document.
Document doc = new Document();
DocumentBuilder builder = new DocumentBuilder(doc);
builder.write("Hello world!");

//OoxmlSaveOptions only applies to Docx, Docm, Dotx, Dotm, or FlatOpc formats.
OoxmlSaveOptions options = new OoxmlSaveOptions(SaveFormat.DOCX);

// Set a password with which the document will be encrypted, and which will be required to open it.
options.setPassword("MyPassword");
doc.save(getArtifactsDir() + "OoxmlSaveOptions.SaveAsDocx.docx", options);

// We will not be able to open this document with Microsoft Word or
// Aspose.Words without providing the correct password.
try
{
    doc = new Document(getArtifactsDir() + "OoxmlSaveOptions.Password.docx"));
}
catch (Exception e)
{
    Assert.assertTrue(e instanceof IncorrectPasswordException);
}
{{< /highlight >}}
