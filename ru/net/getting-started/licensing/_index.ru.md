## Бесплатная пробная версия или временная лицензия

Aspose.Words — это великолепное программное обеспечение, которое разработчики могут опробовать перед покупкой.

### Бесплатная пробная версия

Ознакомительная версия такая же, как и приобретенная: [Пробная версия](https://releases.aspose.com/words/) просто становится лицензируемым, когда вы добавляете несколько строк кода для применения лицензии.

Пробная версия Aspose.Words без указанной лицензии обеспечивает полную функциональность продукта, но вставляет оценочный водяной знак вверху документа при загрузке и сохранении и ограничивает максимальный размер документа несколькими сотнями абзацев.

### Временная лицензия

Если вы хотите протестировать Aspose.Words без ограничений пробной версии, вы также можете запросить 30-дневную временную лицензию. Более подробную информацию можно найти на странице [Получить временную лицензию](https://purchase.aspose.com/temporary-license/).

### Варианты применения лицензии

Лицензии можно применять из разных мест:

* Явный путь
* Папка, содержащая *Aspose.Words.dll*-файл
* Папка, содержащая сборку, вызывающую *Aspose.Words.dll*
* Папка, содержащая сборку записи (ваш *.exe*)
* Встроенный ресурс в сборке называется *Aspose.Words.dll*
* В качестве лимитной лицензии — новый механизм лицензирования

Когда вы ссылаетесь на *Aspose.Words.dll* в приложении, библиотека копируется в выходной каталог (если для параметра *Copy Local* в свойствах этой записи установлено значение false). Часто самый простой способ установить лицензию — поместить файл лицензии в ту же папку, что и *Aspose.Words.dll*, и указать только имя файла без пути.

{{% alert color="primary" %}}

Используйте метод [SetLicense](https://reference.aspose.com/words/ru/net/aspose.words/license/setlicense/#setlicense) для лицензирования компонента.

Вызов **SetLicense** несколько раз не вреден, он просто тратит процессорное время.

Многократный вызов [SetMeteredKey](https://reference.aspose.com/words/ru/net/aspose.words/metered/setmeteredkey/) также не вреден, но просто тратит процессорное время и может привести к неправильному накоплению потребления.

{{% /alert %}}

#### Включите файл лицензии в качестве встроенного ресурса

Удобный способ упаковать лицензию вместе с вашим приложением и гарантировать, что она не будет потеряна, — включить ее в качестве встроенного ресурса в одну из сборок, вызывающих Aspose.Words. Чтобы включить файл в качестве внедренного ресурса, выполните следующие действия:

1. В Visual Studio включите файл .lic в проект с помощью меню "**Файл | Добавить существующий элемент…**".
2. Выберите файл в обозревателе решений и установите "**Создайте действие для встроенного ресурса**" в окне "Свойства".
3. В своем коде вызовите **SetLicense**, который передает только короткое имя файла ресурсов.

#### Применить лимитную лицензию

Aspose.Words позволяет разработчикам применять дозированный ключ. Это новый механизм лицензирования.

Новый механизм лицензирования будет использоваться наряду с существующим методом лицензирования. Те клиенты, которые хотят, чтобы им выставлялись счета на основе использования функций API, могут использовать дозированное лицензирование.

После выполнения всех необходимых действий для получения данного типа лицензии вы получите ключи, а не файл лицензии. Этот дозированный ключ можно применить с помощью специально созданного для этой цели класса [Metered](https://reference.aspose.com/words/ru/net/aspose.words/metered/).

Не вызывайте метод **SetMeteredKey** часто, чтобы этот метод лицензирования правильно накапливал потребление и сообщал об этом нам. Просто создайте экземпляр библиотеки Aspose.Words, один раз вызовите **SetMeteredKey**, затем оставьте экземпляр библиотеки созданным и используйте его повторно.

### Изменение имени файла лицензии

Имя файла лицензии не обязательно должно быть "Aspose.Words.LIC". Вы можете переименовать его по своему вкусу и использовать это имя при настройке лицензии в своем приложении.

### Исключение "Невозможно найти имя файла лицензии"

Когда вы покупаете и загружаете лицензию, веб-сайт Aspose называет файл лицензии *"Aspose.Words.LIC"*. Вы загружаете файл лицензии с помощью браузера. В этом случае некоторые браузеры распознают файл лицензии как XML и добавляют к нему расширение .xml, поэтому полное имя файла на вашем компьютере становится *"Aspose.Words.lic.XML"*.

Когда Microsoft Windows настроен на скрытие расширений для известных типов файлов (к сожалению, в большинстве установок Windows это используется по умолчанию), файл лицензии будет отображаться как *"Aspose.Words. LIC"* в Windows Explorer. Вы, вероятно, подумаете, что это настоящее имя файла, и вызовете **SetLicense**, передав ему *"Aspose.Words.LIC"*, но такого файла нет, отсюда и исключение.

Чтобы решить проблему, переименуйте файл, удалив невидимое расширение .Microsoft. Мы также рекомендуем вам отключить опцию "скрыть расширения" в Microsoft Windows.

## Использование нескольких продуктов Aspose

Если вы используете в своем приложении несколько продуктов Aspose, например Aspose.Words и `Aspose.Cells`, вот несколько полезных советов:

* Установите лицензию для каждого продукта Aspose отдельно. Даже если у вас есть единый файл лицензии для всех компонентов, например, "Aspose.Total.lic", вам все равно придется вызывать **SetLicense** отдельно для каждого продукта Aspose, который вы используете в своем приложении.
* Используйте полное имя класса лицензии. Каждый продукт Aspose имеет **License**-класс в своем собственном пространстве имен. Например, Aspose.Words имеет [Aspose.Words.License](https://reference.aspose.com/words/ru/net/aspose.words/license/), а `Aspose.Cells` — класс `Aspose.Cells`.License. Использование полного имени класса позволяет избежать путаницы в отношении того, какая лицензия к какому продукту применяется.
