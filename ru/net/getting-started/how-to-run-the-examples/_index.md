Чтобы лучше познакомиться с возможностями и функциями Aspose.Words, мы предоставляем примеры, которые можно загрузить из нашего репозитория GitHub, запустить и изучить подробно.

В этой статье вы можете найти системные требования, а также информацию о том, как запускать примеры.

## Требования к программному обеспечению и предварительные условия

Прежде чем загружать и запускать примеры, убедитесь, что вы соответствуете следующим требованиям:

1. Visual Studio-код, Visual Studio 2019.
2. Установлен NuGet Package-менеджер и последняя версия NuGet API для Visual Studio.
3. Выбран параметр **nuget.org** в диалоговом окне "Инструменты → Параметры" в разделе "Диспетчер NuGet Package → Источники пакетов".
4. Активное подключение к Интернету для использования функции автоматического восстановления пакета `NuGet` в проекте "Примеры". Если у вас нет активного подключения к Интернету на компьютере, на котором должны выполняться примеры, проверьте статью [Монтаж](/words/ru/net/installation/) и вручную добавьте ссылку на Aspose.Words.dll в проект "Примеры".

## Загрузите и запустите примеры

Все примеры Aspose.Words для .NET размещаются в формате [GitHub](https://github.com/aspose-words/Aspose.Words-for-.NET). Вы можете клонировать репозиторий с помощью вашего любимого клиента GitHub или загрузить [ZIP-файл](https://github.com/aspose-words/Aspose.Words-for-.NET/archive/master.zip).

Получив копию репозитория, вы можете обнаружить следующее:

- Все примеры находятся в папке **Examples**.
- Существуют файлы решений Visual Studio для C# и Xamarin, созданные в Visual Studio 2019.

Чтобы запустить примеры, откройте файл решения в формате Visual Studio и создайте проект:

- Для примеров **API Reference** структура основана на именах классов, для примеров **Docs** она в основном основана на разделе документации [Руководство разработчика](/words/ru/net/developer-guide/).
- При первом запуске зависимости будут автоматически загружены через NuGet.
- Папка **Data** в корневой папке **Examples** содержит входные файлы, которые использовались в примерах.
- Все примеры можно запускать как модульные тесты.

## Внесите свой вклад в улучшение примеров

Если вы хотите добавить или улучшить пример, мы рекомендуем вам внести свой вклад в проект. Все примеры и демонстрационные проекты в этом репозитории имеют открытый исходный код и могут свободно использоваться в ваших приложениях.

Вы можете разветвить репозиторий, отредактировать исходный код и создать запрос на включение, чтобы внести свой вклад. Мы рассмотрим изменения и включим их в репозиторий, если они окажутся полезными.
