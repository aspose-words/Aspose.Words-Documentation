## Общий

**Вопрос: Что такое Aspose.Words?**

Aspose.Words для .NET — это библиотека классов, которая позволяет вашим приложениям выполнять широкий спектр задач по обработке документов. Используя Aspose.Words для .NET, вы можете создавать, изменять, конвертировать, отображать и печатать документы без использования сторонних приложений.

Подробности можно найти в нашей документации.

------

**Вопрос: Что означает "Aspose.Words не требует сторонних приложений"?**

Для работы Aspose.Words для .NET не требуется установка каких-либо сторонних приложений или дополнительных пакетов программного обеспечения. Просто загрузите и установите Aspose.Words, как описано в статье [Монтаж](/words/ru/net/installation/), и приступайте к работе.

------

**Вопрос: Какие платформы поддерживает Aspose.Words?**

Aspose.Words для .NET охватывает большинство популярных сред разработки и платформ развертывания. Его API можно использовать для разработки приложений для широкого спектра операционных систем, таких как Windows, Linux и Mac OS, а также различных платформ.

Подробную информацию см. на странице [Обзор продукта](/words/ru/net/product-overview/) и в разделе [Платформы и возможности Interop](/words/net/platforms-and-interoperability/).

------

**Вопрос: Какие форматы документов поддерживает Aspose.Words?**

Aspose.Words для .NET поддерживает большинство популярных форматов документов, таких как DOC, DOCX, HTML, Markdown, PDF, XML и другие.

Полный список поддерживаемых форматов можно увидеть на странице [Поддерживаемые форматы документов](/words/ru/net/supported-document-formats/).

## Рендеринг документа

{{% alert color="primary" %}}

Более подробную информацию о рендеринге можно найти в разделе документации [Рендеринг](/words/ru/net/rendering/).

{{% /alert %}}

**Вопрос: Почему макет и шрифты выходного документа отличаются от оригинала?**

Шрифт может быть недоступен, или формат шрифта поддерживается Microsoft Word, но не поддерживается Aspose.Words. Aspose.Words выполнит [Манипулирование и замена шрифтов TrueType](/words/ru/net/manipulating-and-substitution-truetype-fonts/).

Чтобы проверить, отсутствует ли шрифт:

* В Aspose.Words используйте предупреждения о замене шрифта (подробнее см. в разделе "Как распознать, что шрифт был заменен" статьи [Манипулирование и замена шрифтов TrueType](/words/ru/net/manipulating-and-substitution-truetype-fonts/)).
* В Microsoft Word откройте диалоговое окно "Подмена шрифта" (Файл → Параметры → Дополнительно → Подстановка шрифта).

------

**Вопрос: Где следует хранить шрифты для приложений ASP.NET?**

В `APS.NET` нет доступа к папке системных шрифтов со средним уровнем доверия. Пользователи должны хранить шрифты в своей папке.

------

**Вопрос: Почему резервный шрифт не работает на платформах, отличных от Windows, а символы Юникода заменяются глифом .notdef?**

Мы часто можем видеть глиф `.notdef` вместо какого-либо символа Юникода. Обычно это квадрат или знак вопроса.

Это связано с тем, что по умолчанию выбраны резервные настройки Microsoft Office, а шрифты Microsoft Office недоступны. Пользователям необходимо установить шрифты Office Microsoft или изменить резервные настройки.

------

**Вопрос: Почему текст сложного алфавита, например тайского или иврита, по какой-либо причине может отображаться неточно, например из-за неправильного положения диакритических знаков или неправильных лигатур?**

Некоторые сложные рукописные шрифты требуют обработки функций расширенной типографики для их правильного отображения. Расширенная типографика отключена в Aspose.Words по умолчанию. Пользователям необходимо включить расширенную типографику с помощью [TextShaperFactory](https://reference.aspose.com/words/net/aspose.words.layout/layoutoptions/textshaperfactory/).

------

**Вопрос: Нужны ли мне файлы лицензий для шрифтов, которые я использую?**

Да, так и есть. Поэтому при копировании шрифтов имейте в виду, что большинство шрифтов защищены авторским правом. Заранее найдите лицензии на шрифты и убедитесь, что их можно свободно перенести на другой компьютер.

## Вложенный Mail Merge

{{% alert color="primary" %}}

Более подробную информацию о mail merge можно найти в разделе документации [Mail Merge и отчетность](/words/net/mail-merge-and-reporting/).

{{% /alert %}}

**Вопрос: Почему в сгенерированном выводе нет объединенных полей?**

Исходное имя поля слияния остается прежним, не заменяясь необходимыми данными из источника данных, тогда:

* Убедитесь, что данные правильно загружаются в таблицы: правильно задайте [TableName](https://reference.aspose.com/words/net/aspose.words.mailmerging/imailmergedatasource/tablename/) со всеми необходимыми первичными ключами и связями.
* Убедитесь, что поля слияния названы правильно. Используйте метод [GetFieldNames](https://reference.aspose.com/words/net/aspose.words.mailmerging/mailmerge/getfieldnames/), чтобы получить имена всех полей слияния и убедиться, что имена полей слияния в вашем шаблоне совпадают с именем в вашем источнике данных.

------

**Вопрос: Почему в результате вложенного слияния не отображаются данные из дочерней таблицы для первой записи родительской таблицы, а отображаются все элементы последней записи родительской таблицы, даже те, которые на самом деле с ней не связаны?**

Это связано с тем, что регионы слияния в шаблоне сформированы неправильно, из-за чего все вложенные регионы mail merge могут вообще ничего не отображать. Открывающий тег [StartTable](https://reference.aspose.com/words/net/aspose.words/documentbuilder/starttable/) и закрывающий тег [EndTable](https://reference.aspose.com/words/net/aspose.words/documentbuilder/endtable/) должны совпадать и находиться в одной строке или ячейке. Например, если вы начинаете вкладывать области слияния в ячейку таблицы, вы должны закончить область слияния в той же строке, что и первая ячейка.

------

**Вопрос: Почему в каждой записи родительской таблицы отображаются все элементы дочерней таблицы, даже те, которые на самом деле с ней не связаны?**

Это связано с тем, что связи между родительской и дочерней таблицами не настроены или настроены неправильно. Необходимо:

* Обеспечьте целостность данных в **DataSet** и используйте объект **DataRelation** для представления отношений родитель-потомок между связанными таблицами данных.
* См. раздел "Как настроить отношения данных во вложенных Mail Merge с регионами" в статье [Вложенный Mail Merge с регионами](/words/net/nested-mail-merge-with-regions/).

------

**Вопрос: Почему возникает исключение: "System.ArgumentException: это ограничение нельзя включить, поскольку не все значения имеют соответствующие родительские значения" при выполнении вложенного mail merge?**

Это происходит потому, что не каждая родительская запись имеет дочернюю запись, поэтому ваш источник данных не соответствует следующим критериям: *каждая строка в родительской таблице должна иметь связь один-к-одному со строками дочерней таблицы на основе первичной и внешние ключи*.

Отключите ограничения внешнего ключа при создании **DataRelation**.

**Вопрос: Почему содержимое, добавленное в документ, не отображается на той же странице?**

Результат добавления отображается на отдельной странице из-за различий в настройках [PageSetup](https://reference.aspose.com/words/net/aspose.words/pagesetup/) для разделов, в которых документы добавляются вместе. Установите одинаковые настройки **PageSetup** для разделов, в которых документы добавляются вместе.

## Преобразование документа

{{% alert color="primary" %}}

Более подробную информацию о преобразовании документа можно найти в разделе документации [Преобразование документа](/words/ru/net/convert-a-document/).

{{% /alert %}}

**Вопрос: Как конвертировать PDF в Word?**

Это очень просто: просто загрузите документ в модель и сохраните его в любом поддерживаемом формате.

В следующем примере кода показан процесс преобразования файла PDF в DOC:

{{< highlight csharp >}}
// Open the source PDF document
Document pdfDocument = new Document(_dataDir + "PDFToDOC.pdf");

// Save the file into MS document format
pdfDocument.Save(_dataDir + "PDFToDOC_out.doc", SaveFormat.Doc); // .Docx, .Rtf, .WordML, etc.
{{< /highlight >}}

------

**Вопрос: Как конвертировать DOCX в PDF?**

Это также очень просто: просто загрузите документ в модель и сохраните его в любом поддерживаемом формате.

В следующем примере кода показан процесс преобразования файла DOCX в PDF:

{{< highlight csharp >}}
// Open the source DOCX document.
Document doc = new Document(_dataDir + "input.docx");

// Save the file to PDF format.
doc.Save(_dataDir + "output.pdf", SaveFormat.Pdf);
{{< /highlight >}}

## Манипулировать с документом

{{% alert color="primary" %}}

Более подробную информацию о программировании с помощью документов можно найти в разделе документации [Программирование с документами](/words/ru/net/programming-with-documents/).

{{% /alert %}}

**Вопрос: Как разделить документ постранично?**

Aspose.Words позволяет разделить многостраничный документ постранично.

В следующем примере кода показано, как разделить документ и сохранить каждую страницу как отдельный документ:

{{< highlight csharp >}}
Document doc = new Document(MyDir + "Big document.docx");

int pageCount = doc.PageCount;

for (int page = 0; page < pageCount; page++)
{
	// Save each page as a separate document.
	Document extractedPage = doc.ExtractPages(page, 1);
	extractedPage.Save(ArtifactsDir + $"SplitDocument.PageByPage_{page + 1}.docx");
}
{{< /highlight >}}

------

**Вопрос: Как открыть зашифрованный** **документ?**

Вы можете попытаться открыть зашифрованный документ без пароля, что должно привести к исключению.

В следующем примере кода показано, как открыть зашифрованный документ с помощью пароля:

{{< highlight csharp >}}
// Create a document.
Document doc = new Document();
DocumentBuilder builder = new DocumentBuilder(doc);
builder.Write("Hello world!");

//OoxmlSaveOptions only applies to Docx, Docm, Dotx, Dotm, or FlatOpc formats.
OoxmlSaveOptions options = new OoxmlSaveOptions(SaveFormat.Docx);

// Set a password with which the document will be encrypted, and which will be required to open it.
options.Password = "MyPassword";
doc.Save(ArtifactsDir + "OoxmlSaveOptions.SaveAsDocx.docx", options);

// Open the encrypted document by passing the correct password in a `LoadOptions` object.
doc = new Document(ArtifactsDir + "OoxmlSaveOptions.Password.docx", new LoadOptions("MyPassword"));

Assert.AreEqual("Hello world!", doc.GetText().Trim());
{{< /highlight >}}

------

**Вопрос: Как распечатать документ?**

Всего две строчки кода.

В следующем примере кода показано, как распечатать документ двумя способами:

{{< highlight csharp >}}
Document doc = new Document(MyDir + "PrintMe.docx");

// Below are two ways of printing our document.
// 1 -  Print using the default printer:
doc.Print();

// 2 -  Specify a printer that we wish to print the document with by name:
string myPrinter = PrinterSettings.InstalledPrinters[4];

doc.Print(myPrinter);
{{< /highlight >}}

------

**Вопрос: Как редактировать PDF-документ?**

Просто загрузите PDF в модель документа Aspose.Words и внесите изменения.

В следующем примере кода показано, как редактировать документ:

{{< highlight csharp >}}
// Open the source PDF document
Document pdfDocument = new Document(_dataDir + "PDFToDOC.pdf");

DocumentBuilder builder = new DocumentBuilder(pdfDocument);
builder.MoveToDocumentEnd();
builder.Writeln("New paragraph with text");

pdfDocument.Save(_dataDir + "out.pdf", SaveFormat.Pdf);
{{< /highlight >}}
